# 확장 포인트 메모

## 오픈뱅킹

- 현재 `account`, `transaction` 구조를 유지한 채 외부 계좌 동기화 소스 필드 추가 가능
- 수동 입력 거래와 연동 거래를 구분하는 source 메타데이터 계층이 필요
- 중복 거래 식별용 외부 transaction id 저장 지점이 필요

## 영수증 OCR

- 현재 거래 입력 폼 앞단에 OCR 결과를 채워 넣는 구조로 확장 가능
- 이미지 업로드는 Supabase Storage로 연결 가능
- OCR 결과를 바로 저장하지 말고, 사용자 검토 후 거래 생성으로 이어지는 검수 단계가 필요

## 자동 분류

- 현재 `category`, `transaction.description`, `account` 데이터가 기본 학습 신호 역할을 할 수 있음
- 추천 결과는 자동 확정이 아니라 후보 제안 형태로 시작하는 것이 안전
- 잘못된 분류 수정 내역을 피드백 데이터로 쌓을 수 있도록 추후 이벤트 로그가 필요

## 공통 주의점

- MVP 단계에서는 프론트에 비즈니스 규칙을 과도하게 중복하지 않기
- 자동화 기능은 항상 사용자 검토와 취소 경로를 남기기
- 서비스 역할 키를 클라이언트로 노출하지 않기
