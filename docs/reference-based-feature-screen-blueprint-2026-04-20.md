# 레퍼런스 기반 기능/화면 구성도 (2026-04-20)

이 문서는 비개발자도 한눈에 이해할 수 있도록,  
`지금 서비스가 어디까지 왔는지`와 `다음에 어떤 화면/기능을 손봐야 하는지`를 정리한 실행 문서입니다.

## 1) 기준 레퍼런스
- Android: Money Manager, Wallet, Spendee, Monefy, Money Lover
- iOS: Money Lover, YNAB, Goodbudget, Rocket Money, PocketGuard
- 내부 참고 문서:
  - `docs/design-benchmark-playstore.md`
  - `docs/ledger-app-reference-review-2026-04-15.md`

## 2) 제품 목표 (쉽게 요약)
- 기록은 빠르게: 10초 안에 수입/지출 입력
- 분석은 쉽게: 이번 달 상태를 숫자 카드로 즉시 확인
- 관리는 안전하게: 계정/카테고리/예산/반복거래를 한 흐름에서 관리
- 운영은 믿을 수 있게: 반복배치 실행 상태와 실패 원인을 바로 확인

## 3) 핵심 기능 맵 (현재/다음)
| 영역 | 사용자 입장에서 하는 일 | 현재 상태 | 다음 보강 |
|---|---|---|---|
| 빠른 입력 | 거래를 빠르게 등록 | 완료 | 입력 폼 단계 축소(필수 항목만 먼저) |
| 최근 템플릿 | 자주 쓰는 거래를 원탭 재사용 | 완료 | 템플릿 고정(pin), 즐겨찾기 |
| 스타터 팩 | 초반 계정/카테고리 자동 세팅 | 완료 | 사용자 유형별 팩(직장인/프리랜서/자영업) |
| 월간 인사이트 | 예산 초과/지출 비중 확인 | 1차 완료 | 카드 확장(절약 제안, 전월 대비) |
| 거래 검토 | 검색/필터/정렬/일괄처리 | 완료 | 고급 필터 저장, 검색어 추천 |
| 반복거래 운영 | 실행 로그와 실패 사유 확인 | 완료 | 실패 자동 재시도 버튼, 원인 분류 태그 |
| 인증 UX | 로그인/로그아웃 안내 | 개선 중 | 에러 문구/복구 가이드 통합 |
| 모바일 동기화 | 웹과 같은 데이터 즉시 반영 | 완료 | 웹의 스타터 팩/템플릿 UX를 모바일로 이식 |

## 4) 화면 구성도 (Web)
`apps/web/components/ledger-workspace.tsx` 기준

1. 좌측 워크스페이스 네비게이션
- 월별 달력
- 대시보드
- 거래 검토
- 관리
- 설정

2. 메인 콘텐츠 영역
- 대시보드: 요약 카드 + 월간 인사이트 + 위젯 순서 편집
- 달력: 날짜 기반 거래 확인 + 선택 날짜 빠른 입력
- 거래 검토: 카드/테이블 보기, 필터/정렬, 페이징, 내보내기
- 관리: 빠른 입력/계정/카테고리/예산/반복거래/실행로그
- 설정: 프로필/기본 환경값

3. 관리 화면(핵심)
- 빠른 입력: 최근 템플릿 + 스타터 팩 + 거래 입력 폼
- 계정/카테고리/예산/반복거래: 생성-수정-보관 흐름
- 실행로그: 최근 7일/누적 성공·실패 + 상세 사유

## 5) 화면 구성도 (Mobile)
`apps/mobile/src/mobile-ledger-app.tsx`, `apps/mobile/src/screens.tsx` 기준

1. 하단 탭 구조
- 홈
- 입력
- 목록
- 관리

2. 홈 화면
- 이번 달 수입/지출/잔액
- 최근 거래
- 예산 요약
- 반복거래 요약
- 반복거래 실행로그 요약/상세 모달

3. 입력/목록/관리
- 입력: 빠른 금액 칩 + 계정/카테고리 선택 + 저장
- 목록: 거래 수정/삭제
- 관리: 프로필/계정/카테고리/예산/반복거래

## 6) 우선순위 화면 개선안 (디자인 관점)
1. P0: 정보 밀도 재조정
- 대시보드 첫 화면에서 "핵심 3개 숫자"를 가장 크게
- 보조 설명 문구는 줄이고 액션 버튼은 더 명확하게

2. P0: 입력 스트레스 감소
- 빠른 입력에서 기본값 자동 채움 강화
- 최근 템플릿 카드를 더 작고 많이 보이게 개편

3. P1: 관리 화면 피로도 완화
- 섹션 전환 시 맥락 유지(현재 필터/선택값 보존)
- 빈 상태 메시지를 행동 유도형 문구로 통일

4. P1: 모바일-웹 경험 통일
- 모바일에도 스타터 팩 진입점 추가
- 월간 인사이트 축약 카드(Top 3) 추가

5. P2: 운영 신뢰도 체감 강화
- 실행로그에 실패 유형 배지 추가(권한/네트워크/입력값)
- 실패 사유 공유 버튼 위치를 상단으로 올려 즉시 대응

## 7) 사용자 흐름도 (간단 버전)
1. 로그인
2. 스타터 팩 적용(선택)
3. 빠른 입력으로 거래 등록
4. 대시보드에서 월간 상태 확인
5. 거래 검토에서 필터/수정/내보내기
6. 반복거래와 실행로그 확인
